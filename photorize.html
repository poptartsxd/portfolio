Photorize

UX/UI Design, Research, Prototyping | October â€™16 - Present | Personal Project

Photorize is a mobile app that helps photographers find locations to shoot at. Drawing from the experience of other photographers, Photorize guides those who are unfamiliar with an area to discover new places that suit their needs.

My Role: UX/UI Designer & Researcher
Methods: Persona, Wireframe, Interview, Paper Prototyping, User Testing
Tools: Sketch, Paper Prototypes

OVERVIEW

My goal was to help casual photographers find new locations to shoot at. Through my research, I found out that when photographers go to a new location, they would have difficulty finding a place to shoot. My mobile app solves for this user need by displaying recommended locations with sample photos so that users will know where to go.

PROCESS


RESEARCH

Target User Group

My target users are casual photographers. This level of photographers have the following characteristics:

  • Treats photography like a hobby and not a job
  • Likes to post photos on social media
  • Will plan out photoshoots with friends
  • Owns a digital camera (i.e. DSLR)
Interviews
  • Method: semi-structured interviews
  • Who: 3 casual photographers, 2 amateur photographers
  • Goal: to better understand how photographers prepare for a photoshoot
Problem Space

After analyzing my interview data, I discovered the following findings:

  • Now: at a new place, users will oftentimes not know where to shoot and spend time wandering around
  • Future: when planning to go to a new place, users will spend a long time trying to search online for locations to shoot at

DESIGN

Idea Generation

Based on the user needs, the solution should:

  • Now: Allow photographers to see nearby locations when they are at a new place
  • Future: Allow photographers to search for locations when they are planning to go to a new place
  • Future: Allow photographers to save locations for future reference

PROTOTYPE

I went through two iterations of paper prototyping before designing the app in Sketch.

Paper Prototype - Iteration #1

Key features:

  • Map: a map view will display all the potential shooting locations
  • List: a list view will display all the potential shooting locations
  • Add Location: user will be able to add locations that they think are good for shooting
  • My Locations: user will be able to see any locations that they have saved
  • "Good For" Filter: "Good For" will be used to filter certain types of photography locations (i.e. landscape)
  • Trips: user will be able to search and save locations that they want to go to

User Testing #1

I conducted initial user testing with my paper prototypes. Here are my findings:

  • Users were confused about the "Trips" feature. Because there was already a "My Locations" feature, they felt that Trips was redundant
  • Users preferred a map view over list view because they could see how far a place is on the map if they are already near a location
  • Users liked the "Good For" filter but wanted a way to add their own thoughts of what a location is good for
Paper Prototype - Iteration #2

From user feedback, I made the following changes to the next iteration:

  • Removed "Trips" feature and enhanced saved locations feature
  • Removed list view and retained map view
  • Added ability to give input about "Good For"

User Testing #2

Here are my findings from my second paper prototype testing:

  • Users like the "Favorites" feature as it allows them to see their saved places
  • Users wanted to see some location information on the map before going to each individual page
Design Decision #1: Search for locations on a map view

Users can search for locations on a map. These locations are recommended by other photographers.

This allows users to learn about places they have never gone to before.

Design Decision #2: Map view card

I added the ability for users to tap on the location pin to display information about each place. Users can also swipe through the different cards.

This feature allows users to tap through each location easily to see the information.

Design Decision #3: Saved locations feature

Users can save locations that they want to try out or just keep places that they want to remember on this list.

I kept this simple so that users can easily pull up and search through their saved locations.

Design Decision #4: "Good For" filter

Different photographers have different styles of shooting. I added a filter that allows users to look for locations that caters to their preferences.

Design Decision #5: User input for "Good For" filter

Because different locations are suited for different styles of photography, I added the option for users to add what photography type they think the place is good for. This allows for better representation of location information.

Design Decision #6: Location design

Each location will show the top 3 "Good For" types. This makes sure that users will know what the location is most used for.

Users can also add/remove locations from their saved list by tapping the star icon.

Design Decision #7: Displaying photographer's uploaded photos

Each location will also show sample photos that other photographers have uploaded. This allows users to see whether or not a location will match their style.
